Transforming your web.config

When you deploy a Web site, you often want some settings in the deployed application's web.config file to be different from the development web.config file. For example, you might want to disable debug options and change connection strings so that they point to different databases.

Syntax

<element xdt:Transform="operation" xdt:Locator="condition" />

Examples

web.config
<configuration> <connectionStrings> <add name="DB" connectionString="DataSource=(local)/SQLServerInstance;Database=Database;User Id=User;Password=Password;" /> </connectionStrings> </configuration>
dev transformation
<configuration xmlns:xdt="http://schemas.microsoft.com/XML-Document-Transform"> <connectionStrings> <add name="DB" connectionString="DataSource=DevServer/SQLServerInstance;Database=Database;User Id=DevUser;Password=DevPassword;" xdt:Transform="SetAttributes" xdt:Locator="Match(name)" /> </connectionStrings> </configuration>
stage transformation
<configuration xmlns:xdt="http://schemas.microsoft.com/XML-Document-Transform"> <connectionStrings> <add name="DB" connectionString="DataSource=StgServer/SQLServerInstance;Database=Database;User Id=StgUser;Password=StgPassword;" xdt:Transform="SetAttributes" xdt:Locator="Match(name)" /> </connectionStrings> </configuration>
prod transformation
<configuration xmlns:xdt="http://schemas.microsoft.com/XML-Document-Transform"> <connectionStrings> <add name="DB" connectionString="DataSource=ProdServer/SQLServerInstance;Database=Database;User Id=ProdUser;Password=ProdPassword;" xdt:Transform="SetAttributes" xdt:Locator="Match(name)" /> </connectionStrings> </configuration>
